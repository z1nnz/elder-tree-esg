# 綠伴 Adventure V2 設計規格

本輪改版目標不是把既有畫面換一層皮，而是把綠伴整理成同一個自然科技產品語言：App 是城市探索主產品，前台負責讓人一眼理解世界觀，後台負責營運者能快速判斷狀態。

## 核心方向

- 品牌名統一使用「綠伴 Elder Tree」。
- 情緒關鍵字：電影感、自然科技、溫柔冒險、可信任陪伴、少字但有畫面。
- 互動原則：畫面先說故事，文字只補充下一步。
- App 參考地圖探索節奏，但不複製任何 Pokémon GO、皮克敏或其他作品的角色、美術、名稱與玩法規則。
- 前台參考 Nature Beyond、Noomo、Trionn、SOHub 的敘事節奏與互動密度，但不複製版面、素材或文案。

## App Adventure V2

探索頁是一張會回應人的城市地圖：

1. 進入探索頁後直接定位與顯示玩家位置。
2. 沒有「開始探索」主按鈕；任務靠近後自然解鎖。
3. 底部全域 nav 在探索頁降噪，改成綠伴式地圖 HUD 與任務 sheet。
4. 靠近只代表可以接取；完成 SELF_CHECK / TIMER 才讓生命樹成長。
5. 一般 PHOTO_AI 任務維持現有流程；雷達 PHOTO_AI 暫不開放。

## 前台 Cinematic Nature-Tech V2

首頁保留世界樹影片，並分成三段：

1. 初始狀態：世界樹影片、主句、雙 CTA。
2. 第一次滑動：文字淡出，停留世界樹畫面，五個探索點可 hover / tap。
3. 再往下：進入 App 產品故事、城市雷達與台北 3D 任務大屏。

台北 3D 行政區規則：

- 行政區板塊是唯一可互動命中層。
- 裝飾圓圈、光柱、半透明框、熱力區不得吃滑鼠事件。
- 文字資訊集中到右側資料顯示台，避免 3D 投影文字錯位。

## 後台營運中心 V2

後台不追求炫技，追求營運者一眼看懂：

- 首頁是營運總控台，包含 App 實機驗收、雷達任務、PHOTO_AI、待覆核、LINE、裝置狀態。
- 任務管理以工作流呈現：建立、發布、過期、封存、測試任務。
- 所有資料以真實 API 為準，不放假數字、不顯示密鑰或私密環境變數。

## 自製視覺資產規則

- 可使用自製 SVG、AI 圖像、3D / glTF、紋理、粒子與光效。
- 首頁主影片不替換；新資產只作 overlay、section visual、loading fallback 或局部互動。
- 新資產放入清楚目錄：
  - `apps/public-web/public/images/generated/`
  - `apps/public-web/public/models/`
- 手機版優先效能；重資產必須有靜態 fallback。

## Motion 與可及性

- GSAP 動畫需要有 `prefers-reduced-motion` 降級。
- 動畫關閉時內容仍可閱讀，導覽仍可使用。
- 不用快速閃爍、長時間旋轉、過密掃描線。
- 長者友善：按鈕夠大，下一步清楚，錯誤不顯示技術字串。
