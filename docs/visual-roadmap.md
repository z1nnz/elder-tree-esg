# Visual Roadmap：城市生命樹與探索大屏

## 方向

綠伴的視覺不走純寫實，也不走可愛低模，而是「半寫實、溫柔、帶一點魔法感的城市生命樹」。

核心畫面應該形成三段故事：

1. 城市任務大屏：看見城市中安全、可參與的任務熱區。
2. 行動光點回流：使用者完成任務後，光點回到生命樹。
3. 生命樹成長：樹長出新葉、花、果實或季節變化。

## 生命樹 3A 化路線

### MVP 可直接做

- 固定正面構圖，不靠自動旋轉掩飾模型問題。
- 增加葉片密度與樹冠層次。
- 葉片以獨立幾何生成，包含葉脈與微風動畫。
- 枝幹使用曲線管狀幾何，避免木棍感。
- 任務完成光點沿路徑回到樹冠。

### 下一階段

- Bark material：樹皮加入 procedural noise、深淺紋理與粗糙度變化。
- Leaf material：葉片加入漸層、透光、葉脈深度與背光效果。
- Post-processing：加入 bloom、ambient occlusion、柔和景深。
- LOD：手機、平板、桌機使用不同葉片數與陰影品質。
- Growth state：任務完成後可動態新增葉片、花與果實。

### 真正 3A 等級需要的資產

- 手工或 Blender 生成的 GLB/GLTF 主樹模型。
- 低中高三層 LOD。
- 樹皮、葉片的 albedo / normal / roughness / alpha maps。
- Draco / Meshopt 壓縮。
- QA：桌機、手機 Safari、Android Chrome 的 FPS 與記憶體測試。

## 城市探索大屏

城市探索展示區應參考資料視覺化大屏，而不是一般手機地圖。

第一版元素：

- 米白科技面板。
- 立體城市板塊。
- 任務熱力圖：藍綠外圈、黃橘中圈、紅色核心。
- 任務光柱與標籤。
- 側邊 HUD：今日任務熱區、可接取事件、安全定位狀態。

下一階段可加入：

- 任務完成時熱點縮小並產生回流光線。
- 依不同任務分類顯示不同光柱顏色。
- 台北區域輪廓或大安森林公園輪廓的抽象 3D 地形。
- 公開前台只展示公開摘要，不接私人定位資料。

## 工具參考

- `knight-L/sc-datav`：作為 Three.js/R3F 資料大屏、光柱、地圖板塊與 HUD 的方向參考，不整包依賴。
- `juliangarnier/anime`：適合未來處理 DOM/SVG/HUD 數字跳動、進度條與小型 UI 動畫；目前先不加入依賴，避免與 GSAP 重疊。
- `Paidax01/math-curve-loaders`：適合參考數學曲線、掃描線、飛線與 loading 動畫的路徑語言，可用於任務光點回流生命樹。

## 新功能提案

- 生命樹季節：春花、夏葉、秋色、冬光。
- 城市任務天氣：下雨天改為室內/補水/伸展任務。
- 陪伴光點：家人或志工完成陪伴後，樹上出現小燈。
- 實體樹 ambient mode：前台生命樹與實體樹使用同一套顏色語言。
- 城市活動週：社區、協會或校園可發布限時任務季。
