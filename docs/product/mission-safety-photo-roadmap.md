# 綠伴任務池、安全地點與照片辨識任務路線圖

本文件定義 App Adventure Map V3 之後的任務系統推進順序。目標是避免「做一點、停一下、又不知道進度在哪」的狀況；每個階段都有明確完成條件，完成後才進下一階段。

## 目前階段判定

### Phase 1：App 地圖主體驗穩定

完成條件：

- App 登入後直接進入地圖。
- iOS Simulator 沒定位時可 fallback 到大安森林公園。
- 地圖 overlay、附近任務欄、小樹選單不再互相堆疊或 overflow。
- `flutter analyze` 與 `flutter test` 通過。
- PR 已合併回 `main`。

狀態：已完成。

### Phase 2：安全任務池與接取範圍

本階段先做「可控的任務池」，不做真正全台隨機灑點。原因是長者與一般使用者使用時，任務不能被派到危險路口、私人土地、車道中央、工地、偏僻山路或資料品質不明的位置。

完成條件：

- 後台有「台北驗收任務包」與「安全台灣任務池」。
- 任務地點以安全白名單為主，例如公園、廣場、河濱、車站出口、公眾園區。
- 每個任務都有：
  - 座標
  - 接取半徑
  - 任務模式
  - 成長值
  - 安全提醒
  - 陪伴回應模板
- 雷達任務的接取規則保持：進入半徑才 unlock，完成後才加成長值。
- 雷達任務仍只允許 `SELF_CHECK` 與 `TIMER`。
- 後台可一鍵建立並發布任務包。

### Phase 3：照片 AI 任務驗收

Phase 2 完成後再進入照片 AI 實機驗收。照片任務不先接進雷達任務，而是先用一般任務驗證 Evidence + Firebase Storage + Gemini 的流程。

完成條件：

- 一般 PHOTO_AI 任務可拍照。
- App 將照片壓縮後上傳到 Firebase Storage 私有 evidence path。
- 後端透過短效 URL 交給 AI verifier。
- Gemini 回傳：
  - `PASS`：任務完成、生命樹成長。
  - `REVIEW`：進入家人／陪伴者覆核。
  - `FAIL`：不加分，可重新拍攝。
- 同一 Evidence 重送不重複加分。
- PASS / FAIL / 覆核完成後刪除原圖。

### Phase 4：Companion Response 與 LINE 通知

照片 AI 與一般任務完成後，系統將生活片段整理成自然的陪伴回應，而不是問卷或尷尬開場。

完成條件：

- 任務完成後產生自然生活片段。
- 家人頁可看到最近生活片段。
- LINE 推播可帶出自然回覆建議。
- 不問任務或照片中已經看得到的資訊，例如「這是什麼顏色」。

## 安全地點規則

任務池只允許以下類型：

- 公園與綠地。
- 主要廣場與公共開放空間。
- 車站出口周邊的人行空間。
- 河濱或步道，但不得要求靠近水邊、車道或自行車快速通行區。
- 文化園區、校園開放區、社區公共空間。

禁止地點：

- 車道中央、路口中央、施工區。
- 私人住宅、私人店家內部、需要消費才能停留的位置。
- 偏僻山徑、夜間照明不足區域。
- 需要攀爬、跨越圍欄或靠近水域邊緣的位置。
- 任何要求拍攝陌生人、車牌、門牌或敏感資訊的任務。

## 接取半徑規則

雷達任務不是要使用者站在精準座標點上，而是進入圓形接取範圍即可。

- 一般城市廣場：100–140m。
- 公園與河濱：120–150m。
- 車站出口與小型地標：60–100m。
- 最小半徑：25m。
- 最大半徑：150m。

App 行為：

1. 使用者進入半徑。
2. 任務自動 unlock。
3. App 問使用者是否要開始／完成任務。
4. `SELF_CHECK` 由使用者確認完成。
5. `TIMER` 必須等待時間到。
6. 完成後才讓生命樹成長。

## 任務類型規劃

### SELF_CHECK

適合低風險、短時間完成的任務：

- 補水確認。
- 看一眼天空。
- 確認今天舒服步調。
- 找安全座位休息。
- 觀察一棵樹或一片葉子。

### TIMER

適合需要停留一段時間的任務：

- 三分鐘慢呼吸。
- 五分鐘溫和伸展。
- 十分鐘慢走。
- 三分鐘自然聆聽。

### PHOTO_AI

本階段只用於一般任務，不接入雷達任務。

第一批可辨識對象：

- 植物：`leaf`、`plant`、`tree`、`grass`、`flower`。
- 補水：`water bottle`、`bottle`、`cup`、`glass`、`drink`。
- 休息點：`bench`、`chair`、`seat`、`park`。

規則：

- 不辨識人臉、年齡、情緒、身分。
- 不要求拍陌生人。
- 不要求拍住家、車牌、醫療文件或私人物品。
- AI 信心不足時進入 REVIEW，而不是硬判 FAIL。

## 未來全台任務生成方式

未來可以做「看起來像隨機」的全台任務，但不能真正無限制隨機。

建議流程：

1. 建立安全地點白名單。
2. 每個地點標記城市、類型、可停留程度、建議半徑、適合任務類型。
3. 從白名單抽任務，不從任意經緯度抽。
4. 依時間、天氣、日夜、安全風險控制任務出現。
5. 任務過期後自動下架。
6. 管理員可封存或回報不適合地點。

這樣使用者看到的是「任務好像會在城市裡出現」，但營運上仍然安全可控。
