# App V2 今日陪伴驗收手冊

更新日期：2026-07-11

本文件用來驗收 App V2 的第一個主流程：登入後看見「今日陪伴」首頁，完成任務後由真實後端更新生命樹，關閉 App 重啟後狀態仍存在。

## 1. 啟動服務

在專案根目錄啟動 API：

```sh
npm run dev:api:neon
```

如果要驗收 PHOTO_AI，另外啟動 AI verifier，並在 verifier 環境設定 `GEMINI_API_KEY`：

```sh
npm run dev:ai
```

Flutter App 建議用同一個 Wi-Fi 的手機實機驗收。若只驗 UI，可以先用 macOS：

```sh
cd apps/mobile
flutter run -d macos \
  --dart-define=API_URL=http://127.0.0.1:4100/api/v1 \
  --dart-define=MAP_STYLE_URL=https://tiles.openfreemap.org/styles/liberty
```

手機實機請把 `API_URL` 改成開發 Mac 在同一 Wi-Fi 下的區網 IP。

## 2. 今日陪伴首頁

登入後確認首頁出現：

- 目前家庭名稱。
- 生命樹成長值與下一階段進度。
- 下一個推薦行動。
- 任務卡堆疊。
- 家庭訊息或待覆核提醒。
- 最近的城市雷達任務。

若 API 暫時無法連線，App 可顯示離線示範資料，但正式驗收必須使用真實帳號與 Neon 資料。

## 3. 任務驗收

### SELF_CHECK

1. 在首頁或任務頁選一個自我確認任務。
2. 按下完成。
3. 確認提示文字為「生命樹長出新葉 +N」。
4. 重新整理或重啟 App，確認任務仍為已完成，生命樹成長值沒有退回。
5. 重送完成請求或重複操作時，成長值不得再增加一次。

### TIMER

1. 選一個計時任務。
2. 按開始。
3. 時間未達前嘗試完成，App 應提示還需要等待。
4. 時間達成後完成任務。
5. 確認生命樹只增加一次。

### PHOTO_AI

1. 確認首頁或任務卡沒有顯示 Blaze / Storage 尚未開放的舊文案。
2. 選一個 PHOTO_AI 一般任務。
3. 拍植物或水杯。
4. PASS 時：任務完成，生命樹長出新葉。
5. REVIEW 時：任務進入待覆核；提交者不能覆核自己的照片。
6. FAIL 時：不加成長值，App 顯示可重新拍攝。

雷達任務 PHOTO_AI 本階段仍鎖定，不納入驗收。

## 4. 城市探索驗收

1. 進入城市探索頁。
2. 按「開始探索」後才允許 App 使用前景定位。
3. 半徑外不得解鎖雷達任務。
4. 進入半徑後任務狀態變為可接取。
5. 完成雷達任務後，生命樹成長值只增加一次。
6. 停止探索或離開頁面後，App 不應繼續背景追蹤。

macOS 沒有真實 GPS 時，只驗 UI 與受控 demo；定位流程以 iPhone 或 Android 實機為準。

## 5. 重啟與資料持久化

完成上述任務後：

1. 關閉 App。
2. 重新啟動。
3. 登入同一帳號。
4. 確認目前家庭、生命樹、已完成任務、待覆核、家庭訊息與探索狀態仍由 API 恢復。

## 6. 長者友善檢查

- 主要按鈕要清楚，只保留一個最明顯的下一步。
- 錯誤訊息不得顯示 raw exception、stack trace、Storage bucket、Gemini key 或資料庫資訊。
- 技術原因可以轉成操作指引，例如「照片暫時無法安全上傳，請確認網路後再試一次」。
- 任務完成的回饋統一用生命樹語言，不用單純「加分」語言。
